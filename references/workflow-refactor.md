# refactor: 代码重构

## 用法

```
refactor: <重构目标>
refactor: [截图] <重构目标>
```

不改变外部行为，只改内部结构。重构前后功能等价。

> **执行模式说明：** 本工作流中所有「调用 references/xxx.md」均指**读取对应文件，按其中步骤顺序执行**。所有步骤由当前 Agent 顺序完成，不启动子 Agent。

## 阶段 1: 理解重构意图

[阶段 1/8] 理解重构意图...

### 1.1 解析输入

识别重构类型：
- **结构重构**：拆分大组件、提取公共逻辑、调整目录结构
- **性能重构**：优化渲染、减少重复计算、懒加载
- **规范对齐**：Options API → Composition API、class → function 等
- **依赖清理**：移除未使用依赖、统一工具函数

### 1.2 定位代码

根据重构目标搜索相关代码，定位到具体文件和代码位置。

### 1.3 影响分析

列出所有受影响文件和调用链。

**影响 >5 个文件 → ⏸️ 停下等确认**

---

## 阶段 2: 重构方案

[阶段 2/8] 重构方案...

### 2.1 加载框架规范

根据 `.harness-env.json` 中的 `framework` 字段加载对应规范：
- Vue → 读取 `references/vue-best-practices.md`
- React → 读取 `references/react-best-practices.md`

### 2.2 扫描资产

读取 `references/component-scanner.md`，**按需扫描**（scope: targeted）：
- 扫描与重构目标相关的模块

### 2.3 制定方案

- 每步必须原子化（可独立验证）
- 标注每步的功能等价性验证方式
- 标注是否需要中间 commit

### ⏸️ 审批门禁

**影响 >3 个文件时，输出重构方案等用户确认后再动。**

---

## 阶段 3: 实施重构

[阶段 3/8] 实施重构...

1. 按方案逐步执行
2. 遵循框架规范
3. **不引入新功能、不修复旧 bug**（纯结构改动）
4. 每个原子步骤完成后自验：功能行为不变

---

## 阶段 4: 自查

[阶段 4/8] 自查...

读取 `references/code-reviewer.md`，使用 **refactor profile（3 维度）**：
1. 框架规范合规
2. 组件复用（重构后复用度是否提升）
3. 安全性（确认重构没引入新问题、没改变外部行为）

**发现问题 → 立即修复。最多 1 轮。**

---

## 阶段 5: E2E 验证（可选）

[阶段 5/8] E2E 验证...

读取 `references/phases/verify-e2e.md`，传入：
- 变更页面/组件：本次重构涉及的页面
- 验证重点：refactor（功能等价性，重构前后行为一致）

根据验证结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ Playwright 未安装或无法启动 → 自动跳过，继续下一阶段
- ❌ 测试失败或视觉差异 → 回到阶段 3 修复后重新验证

---

## 阶段 6: 代码验证（强制）

[阶段 6/8] 代码验证...

读取 `references/phases/verify-code.md`，传入：
- 工作流类型：refactor
- 变更文件列表：本次任务涉及的所有文件

根据验证报告的总判定决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 验证不完整 → 继续，但在最终输出中标注
- ❌ 未通过 → 修复后重试，连续 3 次失败 → 报告失败详情并停止

---

## 阶段 7: 测试验证

[阶段 7/8] 测试验证...

读取 `references/phases/verify-test.md`，传入：
- 测试范围：全量

根据测试结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 无测试文件 → 跳过，继续下一阶段
- ❌ 测试失败 → 修复后重试

---

## 阶段 8: 重构摘要

[阶段 8/8] 生成重构摘要...

输出：
- **改了什么**：重构涉及的文件和改动
- **为什么改**：重构目的和收益
- **影响范围**：受影响的文件/模块
- **性能变化**（如有）：重构前后的性能对比
- **功能等价性验证**：确认重构没改变外部行为的验证点
