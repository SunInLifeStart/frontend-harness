# bug: Bug 修复

## 用法

```
bug: <bug 描述>
bug: [粘贴截图] <bug 描述>
```

精准定位 + 最小修复，不重构，不做额外优化。

> **执行模式说明：** 本工作流中所有「调用 references/xxx.md」均指**读取对应文件，按其中步骤顺序执行**。所有步骤由当前 Agent 顺序完成，不启动子 Agent。

## 阶段 1: 问题诊断

[阶段 1/7] 问题诊断...

### 1.1 解析 Bug 描述

- 文字描述：提取现象、复现条件、预期行为
- 截图（如有）：分析 UI 异常、报错信息、样式问题
- **有截图时以截图为准**

### 1.2 定位问题代码

1. 根据 bug 描述中的页面/功能关键词搜索代码
2. 搜索 `{{viewsDir}}`、`{{componentsDir}}` 定位页面组件
3. 追踪事件处理链、API 调用链
4. 检查相关 store、composable、util 函数

### 1.3 根因分析

**必须分析根因，不能只处理表象。**

常见前端 bug 根因：
- 数据层：API 返回值结构不匹配、空值未处理、类型错误
- 状态层：响应式丢失、state 更新时序错误、computed 依赖遗漏
- UI 层：CSS 选择器优先级、布局塌陷、z-index 冲突、响应式断点
- 交互层：事件绑定错误、防抖/节流问题、异步竞态
- 权限层：路由守卫遗漏、权限配置缺失

### 1.4 信息不足

如果信息不够定位问题：
- 询问用户：复现步骤、浏览器、设备、报错信息
- 不猜测

## 阶段 2: 修复方案

[阶段 2/7] 修复方案...

### 2.1 加载框架规范

根据 `.harness-env.json` 中的 `framework` 字段加载对应规范：
- Vue → 读取 `references/vue-best-practices.md`
- React → 读取 `references/react-best-practices.md`

### 2.2 设计最小修复

- **只修 bug 本身**，不重构周边代码
- **不做"顺手优化"**
- 如果修复可能影响其他功能 → ⏸️ 告知用户风险，等确认

---

## 阶段 3: 实施修复 + 自查

[阶段 3/7] 实施修复...

### 3.1 实施修复

按方案实施最小修复：
1. 遵循框架规范
2. 不改任何不相关的代码
3. 有 design.md → 修复后的 UI 符合设计规范

### 3.2 自查

读取 `references/code-reviewer.md`，使用 **fix profile（2 维度）**：
1. 框架规范合规
2. 安全性（确认修复没引入新问题）

**发现问题 → 立即修复。最多 1 轮。**

---

## 阶段 4: E2E 验证（可选）

[阶段 4/7] E2E 验证...

读取 `references/phases/verify-e2e.md`，传入：
- 变更页面/组件：本次 bug 修复涉及的页面
- 验证重点：bug（复现场景已修复、关联功能无回归）

根据验证结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ Playwright 未安装或无法启动 → 自动跳过，继续下一阶段
- ❌ 测试失败 → 回到阶段 3 修复后重新验证

---

## 阶段 5: 代码验证（强制）

[阶段 5/7] 代码验证...

读取 `references/phases/verify-code.md`，传入：
- 工作流类型：bug
- 变更文件列表：本次任务涉及的所有文件

根据验证报告的总判定决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 验证不完整 → 继续，但在最终输出中标注
- ❌ 未通过 → 修复后重试，连续 3 次失败 → 报告失败详情并停止

---

## 阶段 6: 测试验证

[阶段 6/7] 测试验证...

读取 `references/phases/verify-test.md`，传入：
- 测试范围：受影响范围

根据测试结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 无测试文件 → 跳过，继续下一阶段
- ❌ 测试失败 → 修复后重试

---

## 阶段 7: 回归确认 + 修复摘要

[阶段 7/7] 生成修复摘要...

### 7.1 Bug 验证

从代码逻辑层面验证：
- bug 根因已消除
- 相关的 UI 状态处理正确
- 没有引入新的问题

### 7.2 修复摘要

输出：
- **根因**：bug 是因为什么导致的
- **修复方式**：改了什么、怎么改的
- **影响范围**：修复影响了哪些文件/功能
- **回归风险**：有没有可能影响其他功能
