# hotfix: 紧急修复

## 用法

```
hotfix: <问题描述>
hotfix: [粘贴截图] <问题描述>
```

生产 P0 级别紧急 bug 的快速修复路径。跳过完整代码审查和全面资产扫描，只做关键检查。

> **执行模式说明：** 本工作流中所有「调用 references/xxx.md」均指**读取对应文件，按其中步骤顺序执行**。所有步骤由当前 Agent 顺序完成，不启动子 Agent。

---

## 阶段 0: 环境预检（简化版）

[阶段 0/7] 环境预检...

1. **读取环境配置** — 检查 `.harness-env.json` 是否存在：
   - 不存在 → 调用 `references/framework-detector.md` 执行环境探测
   - 存在 → 读取缓存
2. **确认 node_modules 存在** — 不存在则提示安装

> **跳过：** 不做 Git 分支检查、不做资产扫描。紧急情况优先速度。

---

## 阶段 1: 快速定位问题

[阶段 1/7] 快速定位问题...

### 1.1 解析错误描述

- 文字描述：提取错误现象、影响范围、紧急程度
- 截图（如有）：分析报错信息、异常 UI
- **有截图时以截图为准**

### 1.2 直接定位

从错误描述直接入手：
1. 根据报错信息 / 页面路径 / 功能关键词搜索代码
2. 搜索 `{{viewsDir}}`、`{{componentsDir}}` 定位目标组件
3. 追踪错误链路（事件 → 函数 → API → 状态）

> **跳过：** 不做全面代码分析，不分析周边模块。只追踪错误链路。

### 1.3 确认根因

- 找到根因 → 进入阶段 2
- 信息不足 → 询问用户：报错堆栈、复现步骤、浏览器控制台信息

---

## 阶段 2: 最小修复

[阶段 2/7] 最小修复...

### 2.1 加载框架规范

根据 `.harness-env.json` 中的 `framework` 字段加载对应规范：
- Vue → 读取 `references/vue-best-practices.md`
- React → 读取 `references/react-best-practices.md`

### 2.2 实施修复

- **仅修复目标问题**，不做任何额外优化
- **不重构**，不改不相关的代码
- **不优化**，不顺手改周边逻辑
- 修改范围越小越好

---

## 阶段 3: 关键检查（3 项）

[阶段 3/7] 关键检查...

修复完成后，只做以下 3 项关键检查：

1. **不引入新 bug** — 修复逻辑正确，没有遗漏边界情况
2. **不破坏构建** — 修改的文件语法正确，import/export 无误
3. **不影响核心流程** — 修复不会导致其他核心功能异常

> **跳过：** 不做完整代码审查（不调用 code-reviewer.md）、不做资产扫描。

---

## 阶段 4: E2E 验证（可选）

[阶段 4/7] E2E 验证...

读取 `references/phases/verify-e2e.md`，传入：
- 变更页面/组件：本次紧急修复涉及的页面
- 验证重点：hotfix（紧急问题已修复、核心流程正常）

根据验证结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ Playwright 未安装或无法启动 → 自动跳过，继续下一阶段
- ❌ 测试失败 → 回到阶段 2 修复后重新验证

---

## 阶段 5: 代码验证（强制）

[阶段 5/7] 代码验证...

读取 `references/phases/verify-code.md`，传入：
- 工作流类型：hotfix
- 变更文件列表：本次任务涉及的所有文件

根据验证报告的总判定决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 验证不完整 → 继续，但在最终输出中标注
- ❌ 未通过 → 修复后重试，连续 3 次失败 → 报告失败详情并停止

---

## 阶段 6: 测试验证

[阶段 6/7] 测试验证...

读取 `references/phases/verify-test.md`，传入：
- 测试范围：最小

根据测试结果决定是否继续：
- ✅ 全部通过 → 继续下一阶段
- ⚠️ 无测试文件 → 跳过，继续下一阶段
- ❌ 测试失败 → 修复后重试

---

## 阶段 7: 修复摘要

[阶段 7/7] 生成修复摘要...

输出：
- **根因**：问题原因
- **修复方式**：改了什么
- **影响范围**：涉及的文件
- **紧急程度**：是否需要立即部署
- **后续建议**：是否需要后续跟进（如需要更完整的修复方案）
